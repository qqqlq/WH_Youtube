# Phase D: AIによる動的動画・画像生成への移行

現在のAVAPは「Pexelsのフリー素材画像」と「テキストのオーバーレイ」によるスライドショー形式の動画を生成していますが、このフェーズでは**「画像生成AI」**や**「動画生成AI」**をAPI経由で統合し、完全にオリジナルのリッチで動きのあるショート動画を自動生成できるようにします。

---

## 🚀 ツールの選定と商用利用について

YouTube等での収益化（商用利用）を前提とする場合、各AIツールの**「商用利用が許可されているか」「公式APIが存在するか（自動化できるか）」**が一番の焦点になります。

### 1. 動画生成AI (Text-to-Video / Image-to-Video)
「nanobanana」等の様々なAIツールがありますが、現在APIとして組み込みやすく、クオリティが高い動画生成AIの候補は以下の通りです。

- **Luma Dream Machine API**
  - **特徴**: 現在最も勢いのある動画生成AIの一つ。公式APIがあり、自動化に最適。
  - **商用利用**: 有料API(クレジット)を利用して生成したものは商用利用可能。
- **Runway Gen-3 (またはGen-2) API**
  - **特徴**: 映像業界でもプロが使う動画AIの先駆者。公式APIあり。
  - **商用利用**: 有料プラン・API経由での生成物は商用利用可能。
- **Kling AI / Minimax (Hailuo) 等**
  - **特徴**: 中国発の高クオリティ動画AI。API対応が進んでいますが、RunwayやLumaに比べると海外での自動化ナレッジが少なめ。

### 2. 画像生成AI (Text-to-Image)
動画生成の元となる高品質な画像を作る、あるいは「わずかにズームするだけの高品質なオリジナル画像」を生成するだけでも、フリー素材とは比較にならない没入感が生まれます。

- **Midjourney**
  - **特徴**: 圧倒的な画質を誇りますが、**公式のAPIが提供されていません**。Discordのハック等で自動化する非公式の手段はありますが、規約上グレーで運用が不安定になりがちです。
  - **商用利用**: 有料プラン（Pro/Mega/Standard等）に加入していれば商用利用可能です。
- **Leonardo.ai API / Stability AI (Stable Diffusion 3) API**
  - **特徴**: Midjourneyに迫る画質（特にLeonardo.ai）を誇り、かつ**開発者向けの公式API**が充実しています。システム組み込みにはこちらが圧倒的におすすめです。
  - **商用利用**: 有料プラン/APIでの生成物は商用利用可能です。

## 🛠 実装のステップ案

### B-1: 収集エージェント(`collector.py`)の拡張
Pexelsを使う「Web検索モード」に加えて、**「AI生成モード」**を追加します。
Gemini（Planner）が生成した `visual_query`（プロンプト）をそのまま画像/動画生成APIに投げ、生成結果のURLをダウンロードしてアセットとします。

### B-2: 動画レンダリング(`editor.py`)の最適化
AIで生成した動画ファイル（通常3〜5秒）をMoviePyで連結します。
現状の「静止画をズームする」エフェクトを外し、生成された動画をそのまま繋ぎ合わせるダイナミックな処理に変更します。

### B-3: フロントエンドでのエンジン選択
音声（VOICEVOX/gTTS）を選択できるようにしたのと同様に、UI上で「素材元（Pexels / Luma API / Leonardo API 等）」を選べるようにします。

## 💡 ご提案・次に決めること
