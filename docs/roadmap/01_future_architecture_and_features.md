# AVAP 将来構想とロードマップ

本書は、動画自動生成パイプライン「AVAP」の今後の本格運用に向けたアーキテクチャ構想と、実装予定の機能ロードマップを記録するものです。

---

## 1. 本格運用アーキテクチャ (Vercel + 自宅サーバー)

現在の Python バックエンドは**動画のレンダリング（MoviePy）**という重い処理を行うため、Vercel のようなサーバーレス環境（実行時間が最大数秒〜数分で制限される）にはデプロイできません。
そのため、**「フロントエンドはクラウド、バックエンドは自宅」** という分離アーキテクチャを採用します。

### 構成図

```mermaid
graph TD
    A[ユーザー (スマホ/PC)] -->|1. UI アクセス| B(Vercel: React フロントエンド)
    B -->|2. API リクエスト| C{Cloudflare Tunnels}
    C -->|3. セキュア通信| D[自宅のミニPC / Ubuntu Server]
    D -->|4. FastAPI + MoviePy 処理| D
```

### デプロイメント詳細
1. **フロントエンド (Vercel)**
   - `frontend/` ディレクトリを GitHub 経由で Vercel にデプロイ。
   - `.env.local` にて `VITE_API_BASE=https://api.yourdomain.com` と設定し、バックエンドへルーティングする。
2. **バックエンド (自宅ミニPC - 例: N100搭載機)**
   - Ubuntu Server をインストールし、FastAPI を systemd サービスなどで常時稼働させる。
   - **Cloudflare Tunnels**（無料）を利用し、安全に自宅のローカルポート(8000)を外部へ公開する。（ルーターのポート開放はセキュリティリスクが高いため非推奨）

## 2. 機能ロードマップ

今後のエンタメ性の向上、動画品質の底上げを目指して以下の機能を順次実装・拡充していきます。

### 2.1 高品質音声エンジンの実装（ずんだもん・複数キャラ対応） (✅ v0.5.0, v0.7.0で実装完了)
- 現状の `gTTS` は機械音声感が強いため、**VOICEVOX (ずんだもん等)** への刷新と、複数種類のボイス（登場人物）をシーンごとに切り替えるUI（一括変更含む）を導入完了。

### 2.2 スライド形式からの脱却（アニメーション・動きの追加） (✅ v0.8.0で本格対応)
- 1枚の画像が静止表示される現状から、視聴維持率を高めるために以下の演出（MoviePy での実装）を追加する。
  - **Ken Burns エフェクト**: (✅ 実装済) ゆっくりとしたズームイン/ズームアウト、左右へのパンニング。
  - **トランジション**: (✅ v0.7.0でCrossFadeInを実装完了) シーン切り替え時のフェード、スワイプ、ズームエフェクト等。
- 画像の上に素材（ミーム画像など）の透過画像を重ねて動かすような高度なオーバーレイ機能 (✅ v0.8.0で位置固定のポン出し合成として実装済)。

### 2.3 効果音 (SE) と BGM の自動配置と合成 (✅ v0.7.0で実装完了)
- **効果音 (SE)**
  - 重要なテロップの出現時や、ツッコミのタイミングに合わせて `audio_catalog.json` 管理下の SE を合成する。
  - AI 台本生成時に `sound_effect` キーを用いた特定効果音の提案機能実装済み。
- **BGM**
  - 台本の雰囲気に合わせたフリーBGM（lofi、cinematic等）をAI側が選定し、音声の裏に音量を下げてループ合成する機能を実装済み。

### 2.4 動画素材ソースの提案・最適化 (✅ v0.8.0で先行実装済)
- Pexels/Pixabay API での画像収集において、現状の静止画だけでなく、縦横比を指定した上での**ショート動画素材（Bロール）**の取得をテスト・導入する (✅ Pexels Video API連携実装済)。
- 企画段階のチャット UI で、AIから「このシーン（ミーム）にはこの画像URLが使えそうです」のように手持ちのミーム素材や特定リンクの事前提示を行う（✅ オーバーレイ画像キーワード連携として実装済）。

### ディレクトリ構造の構想（今後）
本ドキュメントのような「設計書」や「構想」を格納するための専用ディレクトリ `docs/` をプロジェクト直下に設けました。
今後は：
- `docs/architecture/` - システム構成図やインフラ構成案
- `docs/roadmap/` - 各フェーズのタスクリストや要望管理
- `docs/specs/` - API仕様やデータスキーマ設計書

のように分類してナレッジを蓄積していきます。
